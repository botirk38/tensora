Cold-start latency in large language model serving is often reduced to a hardware question: how quickly can storage stream checkpoint bytes? This paper argues that the better abstraction is workload shape. A checkpoint layout determines the file and range requests exposed to the operating system; an I/O backend determines how those requests are submitted and completed; Python bindings and serving engines determine how much of any backend gain reaches the user.

We present Tensora, an open checkpoint-loading framework and measurement harness for evaluating that policy space. Tensora runs synchronous POSIX I/O, Tokio-based async I/O, Linux io_uring, and an adaptive default policy over two layouts: contiguous SafeTensors shards and a partitioned ServerlessLLM-style layout. The evaluation spans Rust backend attribution, Python bindings, a custom vLLM integration, six public checkpoints from 0.6B to 32B parameters, and three environments: an H100 host with local NVMe and working io_uring, plus H200 and A100 container environments where io_uring is blocked.

The results show four regime changes. First, H100 SafeTensors has a scale crossover: sync is fastest at small scales, while io_uring is fastest for 8B--32B multi-shard checkpoints. Second, the ServerlessLLM-style layout changes the ranking: sync is never fastest in the H100 matrix, and async, io_uring, and default read planning trade places depending on partition structure. Third, integration layers reorder but preserve loader impact: Python materialization dominates absolute time, while vLLM cold load_only improves by 37--48% for 8B and 32B models with Tensora's adaptive SafeTensors path relative to vLLM's native loader. Fourth, ordering regimes are more portable than backend identities: small SafeTensors loads favor sync across all three environments, but the mid-to-large-scale winner changes with storage topology and backend availability. The operational conclusion is deliberately bounded: use a format-aware adaptive loader with explicit overrides and capability checks, not sync, async, or io_uring as a universal default by name alone.
