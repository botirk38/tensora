\section{Reproducibility}
\label{app:repro}

This appendix describes how to rebuild the paper and reproduce the ordering evidence. The goal is traceability and comparable regimes, not identical milliseconds on different hardware.

\subsection{Paper build}

Build from \texttt{paper/}:

\begin{verbatim}
pdflatex -interaction=nonstopmode main.tex
bibtex main
pdflatex -interaction=nonstopmode main.tex
pdflatex -interaction=nonstopmode main.tex
\end{verbatim}

The public source bundle should include \texttt{main.tex}, \texttt{template.tex}, \texttt{PRIMEarxiv.sty}, \texttt{sections/}, \texttt{figures/}, \texttt{library.bib}, and \texttt{00README.json}. If a venue template replaces the arXiv style, rebuild with that venue's class and rules.

\subsection{Repository and revision}

The implementation is open source at \url{https://github.com/botirk38/tensora}. Record the exact revision next to exported results:

\begin{verbatim}
git rev-parse HEAD
git status -sb
\end{verbatim}

Also record \texttt{date -u}, \texttt{uname -a}, \texttt{rustc --version}, \texttt{python3 --version}, GPU and driver information from \texttt{nvidia-smi}, and the active vLLM package version when vLLM is used.

\subsection{Rust profiling}

Build the profiler in release mode:

\begin{verbatim}
cargo build --release --bin profile
\end{verbatim}

Run one cell with:

\begin{verbatim}
cargo run --release --bin profile -- <format> <backend> \
  --model-id <MODEL_ID> --iterations 1
\end{verbatim}

\noindent where \texttt{<format>} is \texttt{safetensors} or \texttt{serverlessllm}, and \texttt{<backend>} is \texttt{sync}, \texttt{async}, \texttt{io-uring}, or \texttt{default}. SafeTensors uses the Hugging Face Hub cache. ServerlessLLM-style conversions are stored under the Tensora cache directory, for example \texttt{\$XDG\_CACHE\_HOME/tensora/<slug>/serverlessllm}.

For cold-cache runs on hosts with sufficient privileges, run \texttt{sync} followed by \texttt{echo 3 > /proc/sys/vm/drop\_caches} before each invocation. On restricted containers, use the repository-supported eviction path and report that deviation. The full H100 matrix in \path{results/h100/rust.tsv} was generated by sweeping model ids, formats, and backends with that ritual.

\subsection{Python and vLLM harnesses}

Python benchmarks live under \path{bindings/python/benchmarks/}. The convenience script \path{scripts/run\_benchmarks.sh} runs pytest-benchmark suites and writes JSON outputs. Set \texttt{TENSORA\_BENCH\_MODELS} to one or more Hugging Face model ids, separated by spaces or commas. Set \texttt{TENSORA\_BENCH\_NO\_VLLM=1} when a GPU-backed vLLM environment is unavailable.

When vLLM is enabled, runs are serialized by model to avoid GPU memory interference. The paper's archived exports live under \path{results/h100/}, \path{results/h200/}, and \path{results/a100/}. The H100 vLLM matrix in \path{results/h100/vllm.tsv} sweeps loaders, cache modes, and benchmark kinds: \texttt{load\_only}, \texttt{TTFT}, and \texttt{steady\_state\_decode}.

\subsection{What a replication should check}

A strong replication should check ordering regimes before exact milliseconds:
\begin{itemize}
\item SafeTensors changes backend preference with scale.
\item ServerlessLLM-style partitioning does not follow the SafeTensors sync-first rule.
\item Python overhead is large enough to change observed rankings.
\item vLLM initialization is sensitive to loader choice, while steady-state decode is much less sensitive.
\item Blocking \texttt{io\_uring} changes large-scale winners and exercises fallback behavior.
\end{itemize}

\textbf{Key observations.} (1)~Sync is the slowest or near-slowest ServerlessLLM backend at all Modal scales, replicating the primary H100 finding. (2)~SafeTensors at 0.6B shows a clean separation: async (558\,ms) $<$ default (650\,ms) $<$ sync (713\,ms)---without \texttt{io\_uring}, async becomes the small-scale winner rather than sync. SafeTensors Qwen3-14B: sync [6\,644--19\,466\,ms] vs.\ default [4\,539--28\,174\,ms]---\textbf{OVERLAP}. ServerlessLLM Qwen3-4B: async [3\,612--6\,309\,ms] vs.\ default [3\,587--12\,955\,ms]---\textbf{OVERLAP}. All three confirm that the differences are not statistically separable.

\section{Artifact layout}
\label{app:artifact-layout}

Each results directory contains tab-separated summaries and raw exports. \path{results/<env>/rust.tsv} contains Rust cold-cache profiles. \path{results/<env>/vllm.tsv} contains vLLM integration benchmarks where available. \path{results/<env>/anchor.tsv} contains repeated anchors where collected. \path{results/<env>/raw/} contains pytest-benchmark JSON exports.
