\section{Reproducibility}
\label{app:repro}

This appendix summarises how measurements were produced: what to hold constant, where artefacts live, and which commands reproduce the Rust-layer backend ordering. The goal is traceability and comparable ordering, not byte-identical millisecond reproduction on other hardware.

\subsection{Source bundle and public PDF}

For public distribution, submit \texttt{main.tex}, \texttt{template.tex}, \texttt{PRIMEarxiv.sty}, the \texttt{sections/} tree, \texttt{figures/}, \texttt{library.bib}, and \texttt{00README.json}. Build from \texttt{paper/} with:

\begin{verbatim}
pdflatex -interaction=nonstopmode main.tex
bibtex main
pdflatex -interaction=nonstopmode main.tex
pdflatex -interaction=nonstopmode main.tex
\end{verbatim}

If a conference-supplied document class is adopted later, follow that venue's template and submission rules separately.

\subsection{Repository and revision}

The implementation and harnesses are open source at \url{https://github.com/botirk38/tensora}. The Rust crate \texttt{tensora}, Python package \texttt{tensora\_py}, and benchmark scripts are distributed under the Apache License, Version~2.0. Commands, default output paths, and a minimal environment block are in the repository root \texttt{README.md}. Tabulated timings should correspond to a specific commit hash; record \texttt{git rev-parse HEAD} and \texttt{git status -sb} next to exported results.

\subsection{Rust profiling harness}

Build the profiling binary in release mode:

\begin{verbatim}
cargo build --release --bin profile
\end{verbatim}

Invoke the Rust layer with a public Hugging Face model id:

\begin{verbatim}
cargo run --release --bin profile -- <format> <backend> \
  --model-id <MODEL_ID> --iterations 1
\end{verbatim}

\noindent where \texttt{<format>} is \texttt{safetensors} or \texttt{serverlessllm}, and \texttt{<backend>} is \texttt{sync}, \texttt{async}, \texttt{io-uring}, or \texttt{default}. SafeTensors shards are read from the Hugging Face Hub cache. ServerlessLLM-style conversions are stored under the platform cache directory (for example \texttt{\$XDG\_CACHE\_HOME/tensora/<slug>/serverlessllm}).

Cold-cache runs use \texttt{--iterations 1} with the cache-drop procedure in Section~\ref{sec:experimental_setup}. The published H100 cold matrix (\path{results/h100/rust.tsv}) was produced by looping formats, model ids, and backends with \texttt{sync} and \texttt{echo 3 > /proc/sys/vm/drop\_caches} before each \texttt{profile} invocation. On restricted containers where \texttt{drop\_caches} is unavailable, use the repository-supported cache-eviction flag and record that deviation beside the result.

Five-repetition anchors for selected models (\path{results/h100/anchor.tsv} and \path{results/h200/anchor.tsv} where available) used the same binary and cache-control ritual repeated five times per anchor. Main tables use representative single cold-cache rows for readability.

\subsection{Python and vLLM harnesses}

Python benchmarks call the public binding API under \path{bindings/python/benchmarks/}. The repository provides \path{scripts/run\_benchmarks.sh}, which runs pytest-benchmark suites and writes JSON artefacts under \path{results/benchmarks/} by default. Set \texttt{TENSORA\_BENCH\_MODELS} to one or more Hugging Face model ids, separated by spaces or commas. Set \texttt{TENSORA\_BENCH\_NO\_VLLM=1} to skip vLLM when a GPU-backed serving environment is unavailable.

When vLLM benchmarks are enabled, the script serialises vLLM runs to one model at a time. Archived exports used in the paper appear under \path{results/h100/}, \path{results/h200/}, and \path{results/a100/}. The H100 vLLM matrix in \path{results/h100/vllm.tsv} was produced on a dedicated host by sweeping loaders, model ids, benchmark kinds (\texttt{load\_only}, \texttt{TTFT}, \texttt{steady\_state\_decode}), and cache modes.

\subsection{Environment pins}

Capture at least \texttt{date -u}, \texttt{uname -a}, \texttt{rustc --version}, \texttt{python3 --version}, \texttt{nvidia-smi} GPU and driver output, and the active vLLM package version. Kernel and CUDA versions affect \texttt{io\_uring} behaviour and GPU initialisation. The repository lockfile resolves \texttt{vllm} to 0.11.0 when installed through \texttt{uv}; still record the active environment's package line with exports so a reader can match wheels and CUDA builds.

\subsection{What replication should check}

On different machines, prioritise ordering and regime behaviour: SafeTensors crossover with scale; ServerlessLLM without sync at the front of the ranking; Python overhead large enough to matter; vLLM initialisation sensitive to loader choice while decode is not. Matching every millisecond is unnecessary for validating the main findings.

\subsection{Hub names and cache control}

The main text uses public hub names such as \texttt{Qwen/Qwen3-8B}. Scripts derive filesystem slugs from those ids. Mixing ad hoc checkpoint paths can change shard counts and invalidate comparisons. Prefer hub checksum verification when available.

Dropping the page cache requires \texttt{/proc/sys/vm/drop\_caches}, typically with elevated privileges. The experiments assumed a dedicated host so cache drops do not disturb unrelated tenants. Run \texttt{sync} before \texttt{drop\_caches} so cold reads exercise storage rather than stale clean buffers.

\section{Artefact layout}
\label{app:artefact-layout}

Each environment directory (\path{results/h100/}, \path{results/h200/}, \path{results/a100/}) contains \texttt{rust.tsv} for Rust-layer cold-cache profiles, \texttt{vllm.tsv} for vLLM integration benchmarks where available, \texttt{anchor.tsv} for repetition anchors where available, \texttt{env.txt} for environment metadata where recorded, and \texttt{raw/} for pytest-benchmark JSON artefacts.
