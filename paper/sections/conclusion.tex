\section{Conclusion}
\label{sec:conclusion}

Checkpoint loading for LLM serving exhibits identifiable performance regimes: the optimal I/O backend changes systematically with checkpoint format, model scale, and deployment environment. This paper presents a controlled evaluation of three I/O backends: sync, async, and \texttt{io\_uring}, across two checkpoint layouts, two measured layers, six model scales, and four environments. H100 SafeTensors shifts from sync-favored small loads to \texttt{io\_uring}-favored large loads, with \texttt{io\_uring} providing a 1.28$\times$ advantage at 32B. On H100 ServerlessLLM, sync is never the fastest backend. vLLM integration reorders the absolute rankings relative to Rust but preserves the practical consequence: properly selected backends yield measurable improvements, with \texttt{io\_uring} SafeTensors reducing vLLM cold initialization by 37 to 48\% at 8B and 32B. Cross-environment measurements show that regime structure (small-scale sync dominance, large-scale variation) transfers better than backend identity. These findings suggest that checkpoint loading should be treated as a workload-dependent I/O evaluation problem rather than a fixed initialization cost.
